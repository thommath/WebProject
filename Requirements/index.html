Our Client

  • Client name: PanBot
  • Contact person: Maria Ivashentseva

Our client for this project is a student project that revolves around making a pancake printing robot, PanBot for short. The project is led by two students, Maria Ivashentseva and Thomas Lund Mathisen.

Purpose and goals

The site’s purpose and goals and how they will be accomplished by the site. Try to draw a connection between the goals and the specific content elements of the site.

  • We want to share our passion for 2d-printed pancakes.
  • The purpose of the site is to document our journey to creating a fully fledged pancake printer and also create publicity for our project.
  • To spread the idea, to get sponsors and donations to make new versions, to motivate tired students during the exam period

Target Group

Who is the intended audience or typical user? Describe them and their need for information and other content.

The users of the site will be people that have an interest in printed foods; the physical tools utilized as to make printed foods reality, as well as the programming aspect of the project. By documenting the whole process we can give clear guidance as to how you can set up your own DIY pancake printer, thereby spreading our passion for pancakes.
Other users might just want some designer-pancakes, and we believe we can accommodate their requests too.

Functional Requirements

In this paragraph you'll find the requirements we consider needed to make the website functional for the user. We've seperated the list into two parts; one consisting of the things we consider mandatory, and another of the ones we consider optional, but which will improve our finished product if we manage to comlete them.

Mandatory:
  1. The user must be able to access the newest public blog, vlog or image posts (2-4) from the front page
  2. The user must be able to watch the clients vlogs on the site
  3. The user must be able to access all sites with two clicks
Optional:
  1. The user should be able to make their own pancake designs
  2. The user should be able to give feedback and/or contact client through a form
  3. The user should be able to get bigger versions of the images by clicking the image in the gallery
  4. The user should be able to download, share and/or save their pancake designs
  5. The user should be able to navigate through the site using only keyboard using tab and enter or the arrow keys

Technical Requirements

The technical requirements are the machinery that will make the functional stuff work.

Mandatory:
  1. Built using only js, css and HTML5 (no libraries, frameworks or external css)
  2. Cross-browser /platform support (Firefox, Chrome, Safari - PC and Mac)
Optional:
  1. Statistics and reporting
  2. Mobile support (for advanced smartphones)
  3. Well commented code in functions with more than 5 lines
  4. Scaling website (Scales according to width of browser window)

Maintenance

If known, the server where the site will reside after the project and the names of who will be responsible for the site maintenance.

Communication

The development team and the client will meet weekly to see the progress and quickly correct uncertainties in the agreement. A Scrum method will be used by the developers and be available for the client at any time. The communication besides the weekly meeting will be through email, phone or messenger. Neither party are obliged to work before 10:00 or after 16:00, but should be available for communication during this period.
